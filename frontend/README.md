# واجهات النظام المصرفي الآمن

## 1. مقدمة

يحتوي هذا المجلد على توثيق واجهات المستخدم الخاصة بمشروع النظام المصرفي الآمن.

تم تطوير واجهات النظام باستخدام:

- HTML.
- CSS.
- Bootstrap.
- Jinja2.
- Flask Templates.

لا توجد واجهة منفصلة باستخدام React أو Vue، لأن الواجهات مرتبطة مباشرة بتطبيق Flask.

---

## 2. موقع ملفات الواجهة الفعلية

توجد صفحات HTML داخل:

backend/app/templates/
وتوجد ملفات التنسيق داخل:

backend/app/static/
لذلك لا يتم تكرار ملفات الواجهة داخل مجلد frontend.

الملف:

base.html
هو القالب الرئيسي الذي تعتمد عليه بقية الصفحات.

يحتوي عادة على:

- عنوان الصفحة.
- شريط التنقل.
- روابط ملفات CSS.
- رسائل Flash.
- الجزء المشترك بين الصفحات.
- زر تسجيل الدخول أو الخروج.
- محتوى الصفحة المتغير.

تستخدم الصفحات الأخرى:

{% extends "base.html" %}
ثم تضع المحتوى داخل:

{% block content %}

{% endblock %}

## 17. رسائل النظام

يستخدم المشروع رسائل Flask Flash لإظهار نتيجة العمليات.

مثال نجاح:

flash(
    "Transfer completed successfully.",
    "success"
)
مثال خطأ:

flash(
    "Invalid username or password.",
    "danger"
)
مثال تحذير:

flash(
    "Your account has been disabled.",
    "warning"
)
يتم عرض الرسائل داخل base.html.

---

## 18. ألوان الرسائل

يمكن استخدام أنواع Bootstrap التالية:

| النوع | الاستخدام |
|---|---|
| success | نجاح العملية |
| danger | وجود خطأ |
| warning | رسالة تحذير |
| info | معلومات للمستخدم |

مثال:

<div class="alert alert-success">
    Operation completed successfully.
</div>

## 20. ربط ملف CSS

يتم ربط ملف التنسيق داخل base.html باستخدام:

<link
    rel="stylesheet"
    href="{{ url_for('static', filename='css/style.css') }}"
>
يقوم Flask بإنشاء الرابط الصحيح لملف CSS.

---

## 21. استخدام Bootstrap

يمكن إضافة Bootstrap داخل base.html باستخدام CDN:

<link
    href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
    rel="stylesheet"
>
ويتم إضافة JavaScript في نهاية الصفحة:

<script
    src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js">
</script>

## 23. حماية النماذج

تحتوي النماذج الحساسة على CSRF Token.

مثال:

<input
    type="hidden"
    name="csrf_token"
    value="{{ csrf_token() }}"
>
يستخدم هذا الحقل في صفحات:

- التسجيل.
- تسجيل الدخول.
- التحويل.
- تعديل الملف الشخصي.
- تغيير كلمة المرور.
- وظائف المدير.

---

## 24. استخدام Jinja2

تستخدم قوالب Jinja2 لعرض البيانات الديناميكية.

مثال عرض اسم المستخدم:

<h2>
    Welcome, {{ current_user.username }}
</h2>
مثال عرض الرصيد:

<p>
    Balance: ${{ "%.2f"|format(account.balance) }}
</p>
مثال عرض العمليات:

{% for transaction in transactions %}

    <p>
        {{ transaction.amount }}
    </p>

{% endfor %}
---

## 25. إظهار المحتوى حسب حالة الدخول

مثال:

{% if current_user.is_authenticated %}

    <a href="{{ url_for('auth.logout') }}">
        Logout
    </a>

{% else %}

    <a href="{{ url_for('auth.login') }}">
        Login
    </a>

{% endif %}
---

## 26. إظهار روابط المدير

يمكن إظهار رابط لوحة الإدارة فقط للمدير:

{% if current_user.is_authenticated
      and current_user.role == "admin" %}

    <a href="{{ url_for('admin.admin_dashboard') }}">
        Admin Panel
    </a>

{% endif %}
لكن إخفاء الرابط وحده لا يكفي، ولذلك يتم أيضًا فحص الصلاحية داخل Route.

---

## 32. الخاتمة

تم بناء واجهات النظام المصرفي الآمن باستخدام HTML وCSS وBootstrap وJinja2.

ترتبط الصفحات مباشرة بمسارات Flask، وتعرض بيانات المستخدم والحساب والعمليات بصورة ديناميكية.

توجد ملفات الواجهة الفعلية داخل مجلدات templates وstatic، بينما يستخدم مجلد frontend لتوثيق تصميم الواجهات وتنظيمها.
